Functions

Функции придумана для того, чтобы не повторять один и тот же код в разных местах

Объявление функции

Создание функции происходит (сюрприз) через ее объявление. В начале идет ключевое слово function, после идёт имя функции, а затем список параметров в круглых скобках через запятую, В конце мы открываем тело функции при помощи фигруных скобок, там находится сам код функции.

Функция может быть вызвана по своему имени.

Благодаря функции, код выглядит лучше, и если понадобится поменять сообщение, достаточно будет сделать правку в функции, которая его выводит

Функция и переменные

Стоит учесть,что функция имеет полный доступ к внешним переменным и может изменять их значение, но переменная объявленная внутри функции будет считать локальной, то есть вне функции мы не можем на нее опираться.

если внутри функции создать переменную с названием как у переменной вне функции, то внешняя, просто будет проигнорирована.

Глобальные переменные

Переменные объявленные снаружи всех функций - называются глобальными. глобальные переменные видимы для любой функции (за исключением таких же локальных переменных, потому что они их перекрывают). Желательно сводить использование глобальных переменныъ к минимуму.

Параметры функции

Внутрь функции можно передать любую информацию, используя параметры и аргументы.

  1. параметр - переменная, указанная в круглых скобках в объявлении функции.
  2. аргумент - значение, которое передается функции при её вызове.

Мы можем передать аргумент в параметры, и если глобальная переменная, будет идентична параметру, то на значение первой, это никак не повлияет.

Параметры

Если при вызове функции не указать один из аргументов, то значение этого аргумента будет undefined, для избежания этого, можно задать параметру значение по умолчанию и это не обязательно должна быть строка, можно вписать какое-либо выражение, которое будет выполняться при отстутсвии параметра.

Параметры по умолчанию вычисляются каждый раз, когда функция вызывается без соответствующего параметра. Стоит иметь ввиду, что ранние версии JS не поддерживали параметры по умолчанию, поэтому проверку на undefined производили при помощи конструкции if или при помощи оператора ||. Также, можно использовать оператор нулевого слияния ??. Его использование полезно, если большинство ложных значение, следует расценивать как норальные.

Возврат значения

Функция может вернуть результат, который будет передан в вызвавший ее код

Директива return может находиться в любом месте функции. Как только выполнение дойдет до этого места, функция остановится, и значение возвращается в вызвавший ее код.

Если функция будет пустая, или только с return - она вернет undefined.

Важно: Никогда не добавлять перенос строки между return и его значением, код не выполнится так как интерпретатор JS подставит точку с запитой после return и это будет пустой return. Если мы хотим, чтобы возвращаемое выражение занимало несколько строк, нужно начать его на одной строке вместе с return, либо взять значение после return в скобки

Имя функции

Имя функция должна кратко и максимально точно описывать ее действие, ниже примера удобных начал для названий.

  • get... - возвращает значение

  • check... - что-то проверяет и возвращает

  • calc... - что-то вычисляет

  • create... - что-то создает

  • show... - что-то показывать

Функция должна выполнять только одно действие, которое подразумевается в названии, к примеру функция, которая будет выводить алёрт с возврастом == плохая функция, она должна только вернуть возраст, вызов алерта должен проводиться вне этой функции.
